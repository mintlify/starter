---
title: "Upload a customer document"
openapi: "POST /partner/commenda-os/customers/{customerId}/documents"
---

Upload a file with `multipart/form-data` and create a reusable Commenda customer document in one request.

The created file is scoped to the customer and the response includes `file.fileId`. You may optionally link it to one reusable Commenda OS resource:

- Use `personId` to link the file to a key person.
- Use `businessEntityId` to link the file to a business entity.
- Omit both fields to keep the file customer-scoped only.

Do not send both `personId` and `businessEntityId` in the same request. For incorporation, save the returned `file.fileId`, then upsert it as a typed participant document with `PUT /partner/incorporation/{incorporationId}/participants/{participantId}/documents/{documentKind}`.

The `file` part must include the binary file body. Send `documentName`, `documentSubType`, `personId`, and `businessEntityId` as regular form fields when needed. Commenda infers `fileName`, `mimeType`, and `fileSize` from the uploaded part.

## Example

```bash
curl --request POST \
  --url 'https://api.prod.commenda.io/api/v1/partner/commenda-os/customers/77/documents' \
  --header 'x-api-key: <partner_api_key>' \
  --form 'file=@./passport.pdf;type=application/pdf' \
  --form 'personId=12' \
  --form 'documentName=Passport scan'
```

```json
{
  "file": {
    "fileId": 456,
    "fileName": "passport.pdf",
    "mimeType": "application/pdf",
    "fileSize": 382144,
    "documentSubType": "Other",
    "documentName": "Passport scan",
    "customerId": 77,
    "personId": 12
  }
}
```
