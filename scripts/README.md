# Scripts

This directory contains automation scripts for the documentation.

## update_quickstarts.py

Automatically extracts metadata from quick-start MDX files and updates the `quickStartsData` array in `home.mdx`.

### What it does

1. Scans `docs/get-started/quick-starts/` for `.mdx` files (excluding `home.mdx`)
2. Extracts frontmatter metadata from each file:
   - `title` - Used as the card title
   - `description` - Used as the card description
   - `useCases` - Array of use case tags (e.g., `['real-time analytics', 'AI/ML']`)
   - `products` - Array of product tags (e.g., `['Cloud', 'self-managed']`)
   - `icon` - Optional FontAwesome icon name
3. Generates the file ID from the filename
4. Generates the href from the file path
5. Updates the `quickStartsData` array in `home.mdx`
6. **Updates badges in each quick-start file** between the `{/* AUTOGENERATED_START */}` and `{/* AUTOGENERATED_END */}` markers:
   - Use case badges with blue color
   - Product badges with orange color

### Usage

```bash
python scripts/update_quickstarts.py
```

Or if made executable:

```bash
./scripts/update_quickstarts.py
```

### Quick-start frontmatter format

To ensure your quick-start appears correctly in the explorer, add these fields to your MDX frontmatter:

```yaml
---
title: 'Your Quick Start Title'
description: 'Brief description that will appear on the card'
useCases: ['real-time analytics', 'data warehousing', 'observability', 'AI/ML']
products: ['Cloud', 'self-managed', 'ClickPipes', 'Language clients', 'ClickStack', 'chDB']
icon: 'bolt-lightning'  # Optional FontAwesome icon
---
```

### Autogenerated badges

The script automatically generates badge components in your quick-start files. Add these markers where you want the badges to appear (typically right after the frontmatter):

```mdx
---
title: 'My Quick Start'
useCases: ['real-time analytics']
products: ['Cloud']
---

{/* AUTOGENERATED_START */}
{/* AUTOGENERATED_END */}

## Your content starts here...
```

When you run the script, it will automatically populate the badges:

```mdx
{/* AUTOGENERATED_START */}
<Badge size="lg" color="blue">Real-Time Analytics</Badge>
<Badge size="lg" color="orange">Cloud</Badge>
{/* AUTOGENERATED_END */}
```

#### Available categories

**Use Cases:**
- `real-time analytics`
- `data warehousing`
- `observability`
- `AI/ML`

**Products:**
- `self-managed`
- `Cloud`
- `ClickPipes`
- `Language clients`
- `ClickStack`
- `chDB`

### Example workflow

1. Create a new quick-start MDX file in `docs/get-started/quick-starts/`:
   ```bash
   touch docs/get-started/quick-starts/my-new-quickstart.mdx
   ```

2. Add frontmatter with required fields:
   ```yaml
   ---
   title: 'My New Quick Start'
   description: 'Learn how to do something amazing with ClickHouse'
   useCases: ['real-time analytics']
   products: ['Cloud', 'self-managed']
   icon: 'rocket'
   ---
   ```

3. Run the update script:
   ```bash
   python scripts/update_quickstarts.py
   ```

4. The script will automatically add your quick-start to the explorer page!

### Requirements

- Python 3.6+
- No external dependencies required (uses only standard library)
